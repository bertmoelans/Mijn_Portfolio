Web application security assessment


Samenvatting

Mijn stage is heeft plaatsgevonden bij nFuse. Een bedrijf in de Corda Campus van Hasselt. De volledige titel van mijn stage opdracht luidt als volgt: "Security assessment, exploiting and hardening of extensive used applications". Mijn stage opdracht is dus kort samengevat een PenTest van een kopie van de nFuse website in een afgesloten omgeving. Op deze site heb ik kwetsbaarheden gevonden en onderzocht. Daarna heb ik een mitigatieplan opgesteld voor deze kwetsbaarheden onschadelijk te maken. Zoals je kan zien is de titel in het Engels net zoals de PenTest zelf. Dit is een bewuste keuze door het feit dat in de Cybersecurity wereld Engels wordt gebruikt als taal. De stageopdracht kwam tot stand door een interesse in de OWASP Top 10. Dit is een lijst van beveiligingsrisico's opgesteld door OWASP die het meeste impact kunnen hebben op jouw website. Ik heb hier kennis mee gemaakt in het begin van mijn stage door de juice-shop eens te gebruiken. Dit is een website opgesteld door OWASP waar ik die risico's heb leren kennen en gebruiken. Tijdens mijn stage heb ik meegedaan met dialy standup meetings. Hierin vertelde dan elke werknemer wat hij gedaan had en wat hij zou doen in de toekomst.

Een uitdaging

Het was zeker en vast een uitdaging. In mijn opleiding van Thomas More heb ik veel les gekregen over Cybersecurity. Maar over hoe je gericht een web applicatie moet hacken hadden we nog niet veel gezien. Dat lag dus een beetje uit mijn comfortzone. De meeste tijd van mijn stage was toegewijd aan research en onderzoek van alles wat ik deed. Hierdoor heb ik veel bijgeleerd en zelfstandig leren werken. Daarnaast heb ik het belang van een goede documentatie nogmaals ingezien. Bij een PenTest is het enorm belangrijk om alles goed bij te houden wat je doet. Zo voorkom je dat je zaken meerdere keren hetzelfde uitvoert en kan je leren van eventuele fouten. Tijdens mijn stage heb ik het genoegen gehad om elke vrijdag een presentatie te mogen geven voor de medewerkers van de nFuse. Zo kon ik mijn presentatietechnieken bijschaven en heb ik over het algemeen wat minder stress voor een presentatie te geven.

Bewijsstukken

Je kan hieronder alle bewijsstukken van mijn stage terugvinden. De resultaten van de PenTest werden beschreven in een "Journey" en een "Report" document. In het "Journey" document wordt gedetailleerd besproken wat ik allemaal gedaan heb in chronologische volgorde. Het "Report" document kan gezien worden als een echt PenTest rapport dat ik dit zo zou kunnen afleveren bij een klant. Dus de "Journey" is eerder intern en het "Report" voor extern gebruik.

Project Webdesign

Mijn allereerste IT project!

Mijn eerste echte project was het maken van een website voor een weggeefwinkel genaamd "Het Gouden Hart". Dit was in de projectweek van het eerste semester van de opleiding. We hebben dit in een groep van vier personen gedaan en onder ons het werk verdeeld. Als we ergens moeilijkheden hadden konden we altijd op elkaar rekenen. Het groepswerk verliep vlot, omdat we allemaal even gedreven waren om de opdracht tot een fantastisch einde te brengen.

Mijn aandeel in dit project is vooral de Homepagina en de Welzijnsschakelpagina. Uiteraard heeft iedereen op elke pagina wel iets gedaan, maar dit waren mijn 2 hoofdpagina's. Ik heb hier geleerd hoe je met Materialize een website kan bouwen. Dit is een tool waarin al code verwerkt is, die je dan kan gebruiken op je eigen website. Deze materialize code kan nog aangepast worden en dat hebben we zeker gedaan. Het was dus niet gewoon kopiëren en plakken. Daardoor zijn mijn webskills enorm gegroeid op enkel een week tijd. Ik ben hier heel dankbaar voor. Hieronder vindt u enkele foto's van de site in een slideshow gegoten.

Project Systems Engineering & Analysis

Oplossend denkvermogen geactiveerd!

Het tweede project dat ik graag wil bespreken is een project van het vak Systems engineering and analysis. Een vak dat gegeven werd door Eric Pauwels (leerkracht) in samenspraak met Dirk Verbeek (projectleider/klant). We hebben geleerd hoe we flexibel een project doorlopen in team en met welke hulpmiddelen zoals een scrum-board. Je bent natuurlijk niet flexibel als je niet snel aanpassingen kan doorvoeren in opdracht van de klant. We leerde snel te anticiperen op de veranderende mening van de klant. Wat me vooral is bijgebleven is dat je voldoende tijd moet nemen voor je effectief aan de praktijk begint. Een goed doordachte planning en takenverdeling is essentieel voor een project te beginnen. Ook is het belangrijk om in stappen te werken waarbij elke stap volledig is. Eerst bouw je een fiets en daarna pas een brommer.

Het project zelf ging over het probleem van overtollige regenval in België en het gebrek aan opvang hiervoor. Zo zijn we aan de slag gegaan door te brainstormen en hebben we een systeem bedacht dat de regenputten in ieders tuin verbindt. Dit systeem is remote bestuurbaar en elke put wordt automatisch op hetzelfde niveau gebracht als dat van zijn buren. Ik zelf heb me vooral bezig gehouden met Excel. Hierin wilde ik de effectieve werking van het pompsysteem vastleggen en visueel duidelijk maken. Voor de eerste keer in mijn leven heb ik leren met macro's op Excel. Dit was een uitdaging maar uiteindelijk wel een geslaagde uitdaging naar mijn mening. Het Excel bestand kan u downloaden door op volgende link te klikken Project_SE&A.xlsm


Project Hosting

Mijn eerste hosting project!

Het uiteindelijke doel van dit project was om een hosting service aan te bieden voor PHP-projecten. Dit werd mogelijk gemaakt door een Vsphere omgeving en een OpenVPN connectie. Het was een grote uitdaging waar ik enorm veel van heb geleerd. We hebben een netwerkinfrastructuur opgesteld met een Firewall die al het verkeer filtert van buiten naar binnen en visa versa. We hebben de Firewall geconfigureerd met PfSense. Daar hebben mijn collega Vincent Herroelen en ik voor een groot deel van het project aan gewerkt. Deze netwerkinfrastructuur moest stabiel zijn en effectief werken. Dat is natuurlijk de basis van het project. Het was een grote verantwoordelijkheid die we tot een heel goed einde hebben gebracht. Daarnaast heb ik veel gewerkt aan de documentatie en planning.

We werkten in een team van zeven personen en hebben zo veel software kunnen implementeren op onze interne virtuele machines zoals Zabbix voor monitoring, OsTicket als helpdesk systeem, Varnish om de snelheid van de website te optimaliseren enzovoort. Hieronder vindt u de topologie van ons netwerk. Als u graag onze documentatie doorneemt klik dan gerust op de topologie of download ons Project_Verslag.pdf

Project 4.0

Een project met teamleden van alle afstudeerrichtingen!

In eerste instantie werden we samen geroepen door de stad Herentals om een Smart City oplossing te bedenken. Herentals zou graag Smart City projecten willen starten om zo de stad smarter te maken. Mijn team genaamd Connect IT heeft een slimme oplossing gevonden voor het vuilnis ophaalsysteem. We ontwikkelden een web applicatie met een dashboard waarop een overzicht te zien is van alle vuilnisbakken in Herentals. Daarbij kan je zien hoe vol een vuilnisbak is, binnen hoeveel dagen het moet opgehaald worden enzoverder. We zijn agile te werk gegaan en op het einde van elke sprint hebben we de feedback van de opdrachtgever verwerkt in onze applicatie. Bekijk zeker ons filmpje eens op YouTube!

Connect IT is een team van zes personen inclusief mezelf waarbij we gewerkt hebben met Angular, een .Net framework, Qlik voor de visualisatie, een live IoT opstelling en Azure om alles met elkaar te verbinden op netwerk niveau. De site zelf wordt gehost op Netlify. Ik was verantwoordelijk voor de Azure omgeving waar IoT data automatisch verstuurd kan worden naar de site. Ook het onderhouden van de database was een grote taak door het gebruiken van SQl commando's.
